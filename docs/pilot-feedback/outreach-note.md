# Pilot Outreach Note

Use this only after explicit approval.

## Short Version

```text
Hi, I am building OSS Intake Doctor, a no-cost local toolkit that helps maintainers improve GitHub issue intake before enabling public bot comments or heavier automation.

It audits issue forms, checks sample issues in dry-run mode, and gives maintainers a short summary of missing details, possible support routing, security-sensitive routing, and duplicate risk.

There are no paid APIs and no external issue-content sharing in the core workflow. Would a 5-minute dry-run review of your issue templates be useful?
```

## Longer Version

```text
Hi, I am building OSS Intake Doctor for open-source maintainers.

The project focuses on the intake layer: better issue forms, clearer bug-report requirements, support/security routing, and dry-run issue analysis before maintainers trust public automation.

The core workflow is local and no-cost:
- no external APIs
- no required paid services
- no public comments by default
- no labels by default
- no auto-close behavior

I am looking for maintainer feedback on whether this would reduce follow-up work in real issue queues. If useful, I can run through the README, issue-template audit, and sample report and record only a short public-safe feedback note.
```
