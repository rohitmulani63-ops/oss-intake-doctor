# Pilot Feedback Kit

Use this folder to collect maintainer feedback once OSS Intake Doctor is public and a maintainer has reviewed or tried it.

The feedback should be short, factual, and safe to publish. Do not include private issue text, secrets, personal contact details, or anything copied from a private repository.

## What Counts As Pilot Evidence

Good pilot evidence can be:

- A maintainer saying the issue-template audit is useful.
- A maintainer trying the local commands and sharing friction.
- A maintainer comparing one of their issue templates with the gallery.
- A maintainer asking for a clearer rule, label, or output format.
- A maintainer declining the tool but explaining what would make it useful.

## What To Capture

Use [template.md](template.md) for each pilot note.

Minimum fields:

- Date.
- Repository or project type.
- Maintainer role.
- What they reviewed or ran.
- What helped.
- What was confusing.
- Follow-up idea.

## Safe Workflow

1. Ask for permission before recording public feedback.
2. Keep the note short and neutral.
3. Remove private details.
4. Link only public repositories or public comments.
5. Add one maintainer-useful follow-up item.

## No-Cost Boundary

Pilot work should stay inside:

- Local commands.
- Public GitHub repo features.
- Manual maintainer review.
- No API keys.
- No paid services.
- No hosted data collection.
