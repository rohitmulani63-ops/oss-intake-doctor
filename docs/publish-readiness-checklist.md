# Publish Readiness Checklist

Use this checklist before publishing the repository.

## Required

- [ ] README explains the problem and product clearly.
- [ ] README includes no-cost, no-external-API positioning.
- [ ] README does not claim this is the only GitHub issue triage tool.
- [ ] `LICENSE` exists.
- [ ] `CONTRIBUTING.md` exists.
- [ ] `SECURITY.md` exists.
- [ ] `CODE_OF_CONDUCT.md` exists.
- [ ] `NO_COST_POLICY.md` exists.
- [ ] `docs/positioning.md` exists.
- [ ] `docs/issue-form-doctor.md` exists.
- [ ] `docs/repro-pack-plan.md` exists.
- [ ] `docs/phase-1-proof-pack-2026-06-17.md` exists.
- [ ] `docs/openai-oss-application-draft.md` exists.
- [ ] `docs/public-repo-template-benchmark-2026-06-17.md` exists.
- [ ] `docs/github-research-refresh-2026-06-17.md` exists.
- [ ] `docs/verification-log-2026-06-18.md` exists.
- [ ] `docs/verification-log-2026-06-17.md` exists.
- [ ] Competitive research is linked from the README.
- [ ] Trust and safety docs exist.
- [ ] GitHub issue templates exist.
- [ ] Dry-run workflow example exists.
- [ ] Tests pass locally.
- [ ] Demo runs locally.
- [ ] Local issue-bank report runs.
- [ ] Local issue-template audit runs.
- [ ] No dependencies are added accidentally.
- [ ] No API or network client code exists.

## Before Public Launch

- [ ] Replace placeholder issue-template links in `.github/ISSUE_TEMPLATE/config.yml`.
- [ ] Decide final repo owner/name.
- [ ] Decide whether MIT license is final.
- [ ] Add maintainer contact in `SECURITY.md`.
- [ ] Add real screenshots or workflow-log snippets after first public test.
- [ ] Confirm no private files are included.
- [ ] Confirm docs say dry-run first, comment-only later.
- [ ] Confirm OpenAI OSS readiness is framed as a later evidence-backed step.

## Do Not Do Without Explicit Approval

- [ ] Publish the repository.
- [ ] Contact maintainers.
- [ ] Apply to programs.
- [ ] Add paid APIs.
- [ ] Deploy hosting.
- [ ] Add dependencies with cost or security risk.
