# Code of Conduct

OSS Intake Doctor is for maintainers and contributors who want healthier open-source issue queues.

## Expected Behavior

- Be respectful.
- Assume good intent where reasonable.
- Focus feedback on the work, not the person.
- Keep criticism specific and useful.
- Respect maintainers' time.
- Protect sensitive security information.

## Unacceptable Behavior

- Harassment or personal attacks.
- Publicly posting exploit details for active vulnerabilities.
- Spam, trolling, or repeated low-effort disruption.
- Discrimination or abusive language.
- Pressuring maintainers to take unsafe actions.

## Project-Specific Principle

This project does not shame contributors.

The tool should improve report quality and maintainer signal without accusing people of being lazy, malicious, or AI-generated.

## Enforcement

Before this repository is published, enforcement is handled by the project owner.

After publishing, this file should be updated with maintainers and reporting contacts.
